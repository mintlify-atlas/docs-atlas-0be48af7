Vale is a command-line tool that brings code-like linting to prose. It's **fast**, **cross-platform** (Windows, macOS, and Linux), and **highly customizable** — letting you enforce any style guide, from major editorial standards to your own in-house rules.

## Key features

- **Markup support**: Vale understands Markdown, AsciiDoc, reStructuredText, HTML, DITA, Org, MDX, and plain text at a structural level — not just as raw strings.
- **Extensible YAML rule system**: Author style rules using 12 built-in extension points (existence, substitution, capitalization, spelling, and more) without writing any code.
- **Standalone binary**: Download and run immediately. No package manager or language runtime needed.
- **Cross-platform**: Native builds for Windows, macOS (Intel and Apple Silicon), and Linux (amd64 and arm64).
- **Vocabulary management**: Define accepted and rejected word lists per project to keep terminology consistent.
- **Tengo scripting**: Write script-based rules using the Tengo language for logic that YAML alone can't express.
- **Hunspell spelling**: Drop in standard Hunspell dictionaries for accurate, locale-aware spell checking.
- **CI-ready output**: Choose from CLI, JSON, or line-based output to slot into any automation workflow.

## Vale vs. other tools

All tools in the table below are open-source, CLI-based prose linters. Implementations and feature sets vary significantly.

| Tool | Extensible | Checks | Supports markup | Built with | License |
|------|------------|--------|-----------------|------------|---------|
| **Vale** | Yes (YAML) | spelling, style | Yes (Markdown, AsciiDoc, RST, HTML, XML, Org) | Go | MIT |
| textlint | Yes (JavaScript) | spelling, style | Yes (Markdown, AsciiDoc, RST, HTML, Re:VIEW) | JavaScript | MIT |
| RedPen | Yes (Java) | spelling, style | Yes (Markdown, AsciiDoc, RST, Textile, Re:VIEW, LaTeX) | Java | Apache-2.0 |
| write-good | Yes (JavaScript) | style | No | JavaScript | MIT |
| proselint | No | style | No | Python | BSD 3-Clause |
| Joblint | No | style | No | JavaScript | MIT |
| alex | No | style | Yes (Markdown only) | JavaScript | MIT |

"Supports markup" means the format is understood structurally — for example, code blocks are excluded from spell check rather than treated as plain text.

"Extensible" means there is a built-in way to write custom rules without modifying the tool's source code.
