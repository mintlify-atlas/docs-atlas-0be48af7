## Why Vale?

Vale understands **markup** — it doesn't just scan raw text. It parses Markdown, AsciiDoc, reStructuredText, HTML, DITA, and more, so it avoids false positives inside code blocks, headings, and other structural elements.

## Get started in three steps

<Steps>
  <Step title="Install Vale">
    Download the binary for your platform via Homebrew, Chocolatey, or direct download:

    ```bash
    brew install vale
    ```
  </Step>
  <Step title="Create a configuration file">
    Add a `.vale.ini` to your project root:

    ```ini .vale.ini
    StylesPath = .vale/styles

    MinAlertLevel = suggestion

    Packages = Microsoft

    [*.md]
    BasedOnStyles = Vale, Microsoft
    ```
  </Step>
  <Step title="Sync and lint">
    Download your style packages and run Vale on your docs:

    ```bash
    vale sync
    vale docs/
    ```
  </Step>
</Steps>

## Supported formats

Vale understands structure across many formats, applying rules only where they make sense:

| Format | Extensions |
|--------|------------|
| Markdown | `.md`, `.mdown`, `.markdown`, `.markdn` |
| AsciiDoc | `.adoc`, `.asciidoc`, `.asc` |
| reStructuredText | `.rst`, `.rest` |
| HTML | `.html`, `.htm`, `.shtml`, `.xhtml` |
| DITA | `.dita` |
| Org | `.org` |
| MDX | `.mdx` |
| Plain text | `.txt` |

<Note>
  Vale also lints prose inside source code comments for Go, Python, JavaScript, TypeScript, Ruby, Rust, and many more languages.
</Note>
